## 2024-05-24 - POSIX Option Bundling Bypass in Command Blocklist
**Vulnerability:** A blocklist validating command-line arguments for a subprocess (ilspycmd) used `StartsWith` and `Equals` to block specific dangerous options (e.g. `-o`, `-p`). However, it failed to block bundled POSIX short options, where a short option is concatenated directly with its value (e.g., `-otmp/path`). Because `-otmp/path` does not exactly equal `-o` and does not start with `-o:` or `-o=`, it bypassed the validation, allowing arbitrary file writes.
**Learning:** Validating POSIX-compliant command arguments requires separating the option key from its bundled value before comparing. Checking for string equality on the whole argument is insufficient because single-character options implicitly absorb the rest of the string as their argument value.
**Prevention:** Explicitly parse arguments by matching just the first character against known dangerous short options, being careful to whitelist any explicitly safe long options that happen to start with that character.
