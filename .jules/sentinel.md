## 2024-05-23 - Command Injection Risk in RunRawCmd
**Vulnerability:** The `RunRawCmd` tool parses arbitrary arguments and directly executes `ilspycmd` with those arguments, providing a command injection risk or unintended file manipulation risk. While it doesn't use a shell and thus prevents standard shell injection (`&&`, `|`), it allows arbitrary arguments to be passed to `ilspycmd` which could have side effects or write files locally. Furthermore, the argument parsing loop does not sanitize inputs securely.
**Learning:** We need to be careful when building arguments to be sent to a command. Passing arbitrary arguments directly without proper sanitization/allowlisting to `Process.Start` can be dangerous if the target application (`ilspycmd`) supports destructive features (e.g. writing arbitrary files to disk if given path traversal arguments).
**Prevention:** As Sentinel, the `raw_cmd` tool itself exposes arbitrary argument passing by design, so to properly secure it, we should validate the arguments against a known safe allowlist, or restrict potentially dangerous flags (like `-o` for output dir).
