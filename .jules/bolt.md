## 2025-05-18 - Prevent Redundant Subprocess Executions via Proper Caching
**Learning:** Found that some metadata commands (`TypeInfo`, `ListMembers`) were redundantly executing the `ilspycmd` subprocess even when the decompiled code was already available in `CachedAssembly.Decompiled`. The C# worker uses caching (like `CachedAssembly.Decompiled` and `CachedAssembly.Members`) specifically to minimize expensive external OS-level calls, and these code paths were missing checks to read from the decompilation cache before falling back to `IlspyCmd()`.
**Action:** Always check existing caches (`CachedAssembly.Decompiled`, etc.) before initiating expensive OS-level subprocess executions (`IlspyCmd`) for operations that rely on the same underlying decompiled data. Ensure new endpoints or metadata commands integrate properly with the cache lifecycle.
